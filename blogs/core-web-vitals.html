What Are Core Web Vitals?

Core Web Vitals are real-world performance metrics Google uses to measure the quality of a user's experience on a webpage. They directly influence search rankings on both mobile and desktop.

The Three Core Metrics

LCP — Largest Contentful Paint

How long the largest visible element takes to fully render. The closest proxy for "how fast does this page feel?"

  • Good: under 2.5 seconds
  • Needs improvement: 2.5s – 4.0s
  • Poor: over 4.0 seconds

INP — Interaction to Next Paint

Replaced FID in March 2024. Measures overall responsiveness to user input throughout the entire visit.

  • Good: under 200 milliseconds
  • Needs improvement: 200ms – 500ms
  • Poor: over 500 milliseconds

CLS — Cumulative Layout Shift

How much visible content unexpectedly shifts while the page loads.

  • Good: 0.1 or less
  • Needs improvement: 0.1 – 0.25
  • Poor: above 0.25

How to Measure Your Scores

  • Google Search Console — real-user data under "Core Web Vitals" in the Experience section
  • PageSpeed Insights — analyses a single URL with lab and field data plus recommendations
  • Chrome DevTools — measure performance directly in your browser

Always prioritise field data (real user measurements) over lab data.

How to Improve Your Scores

Improving LCP

  • Use a fast host and enable a CDN
  • Preload your LCP image using <link rel="preload">
  • Remove render-blocking CSS or JS in the <head>
  • Compress and serve images in WebP format

Improving INP

  • Defer or remove non-essential JavaScript
  • Break up long JavaScript tasks into smaller chunks
  • Minimise third-party scripts

Improving CLS

  • Set explicit width and height on all images and videos
  • Reserve space for ads and embeds before they load
  • Avoid injecting content above existing content after page load

Why It Matters for SEO

Pages passing Core Web Vitals thresholds earn Google's "Good page experience" designation. While not the dominant ranking factor, they act as a tiebreaker between pages of similar quality — and poor performance damages user experience and conversion rates regardless of rankings.

Frequently Asked Questions

Common questions about Core Web Vitals, answered clearly.

Do Core Web Vitals directly affect Google rankings?+

Yes, but they are not the dominant ranking signal. Google uses Core Web Vitals as part of its Page Experience ranking system — they act as a tiebreaker when two pages have similar content quality and authority.

Fixing poor scores is more likely to improve conversion rates and reduce bounce rates than to cause dramatic ranking jumps on its own.

What is a good LCP score and how do I check mine?+

A good LCP score is under 2.5 seconds. Scores between 2.5s and 4.0s need improvement, and anything above 4.0s is considered poor.

To check yours: visit PageSpeed Insights (pagespeed.web.dev), enter your URL, and look at the LCP value. Google Search Console also shows LCP data across all your pages under the Core Web Vitals report.

What is the difference between INP and the old FID metric?+

FID only measured the delay before the browser began processing the very first interaction on a page — and only captured a single interaction.

INP, which replaced FID in March 2024, measures the full latency of all interactions throughout a visit and reports the worst-case result. It is a far more comprehensive measure of overall responsiveness.

Why does my CLS score keep changing between tests?+

CLS varies between tests because it depends on which resources load on any given visit — fonts, images, ads, embeds — and how quickly they load relative to the rest of the page.

For a reliable picture, rely on field data from Google Search Console, which aggregates real user measurements over a 28-day window, rather than single lab tests.

Does improving Core Web Vitals help on mobile more than desktop?+

Core Web Vitals are measured separately for mobile and desktop in Search Console. Mobile scores are typically harder to achieve because mobile devices have less processing power and are often on slower connections.

Since Google uses mobile-first indexing, your mobile scores carry more weight. Prioritise mobile performance improvements first.

Can third-party scripts cause poor Core Web Vitals scores?+

Yes — third-party scripts are one of the most common causes of poor Core Web Vitals, particularly for INP and LCP. Chat widgets, advertising scripts, analytics tools and tag managers all add JavaScript that competes for the browser's main thread.

To minimise their impact: load third-party scripts with async or defer, audit your scripts regularly, and remove any you no longer need.
