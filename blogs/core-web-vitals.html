Core Web Vitals: A Complete Guide

Published May 2026 4 min read

What Are Core Web Vitals?

Core Web Vitals are a set of real-world performance metrics that Google uses to measure the quality of a user's experience on a webpage. They are part of Google's broader Page Experience signals and directly influence search rankings on both mobile and desktop.

They are designed to capture what users actually feel when they visit a page — how fast it loads, how quickly it responds to interaction and how stable the layout is while loading.

The Three Core Metrics

LCP — Largest Contentful Paint

LCP measures how long it takes for the largest visible element on the page (usually a hero image or a large heading) to fully render. It is the closest proxy for "how fast does this page feel?"

  • Good: under 2.5 seconds
  • Needs improvement: 2.5s – 4.0s
  • Poor: over 4.0 seconds

INP — Interaction to Next Paint

INP replaced FID (First Input Delay) in March 2024. It measures the overall responsiveness of a page to user input — clicks, taps and key presses — throughout the entire visit, not just the first interaction.

  • Good: under 200 milliseconds
  • Needs improvement: 200ms – 500ms
  • Poor: over 500 milliseconds

CLS — Cumulative Layout Shift

CLS measures how much the visible content unexpectedly shifts around while the page loads. Unexpected movement — caused by images loading without reserved space, or ads injecting above content — creates a frustrating experience and lowers your CLS score.

  • Good: 0.1 or less
  • Needs improvement: 0.1 – 0.25
  • Poor: above 0.25

How to Measure Your Scores

Google provides several free tools to assess Core Web Vitals:

  • Google Search Console — shows real-user data for your entire site under "Core Web Vitals" in the Experience section
  • PageSpeed Insights — analyses a single URL and provides both lab and field data, plus specific recommendations
  • Chrome DevTools — measure performance directly in your browser during development

Always prioritise field data (real user measurements) over lab data. Field data reflects genuine user conditions; lab data is a controlled simulation.

How to Improve Your Scores

Improving LCP

  • Use a fast web host and enable a CDN (Content Delivery Network)
  • Preload your LCP image using <link rel="preload">
  • Remove render-blocking resources (large CSS or JS files in the <head>)
  • Compress and serve images in WebP format

Improving INP

  • Defer or remove non-essential JavaScript
  • Break up long JavaScript tasks into smaller chunks
  • Minimise third-party scripts (chat widgets, analytics, ad tags)

Improving CLS

  • Always set explicit width and height attributes on images and videos
  • Reserve space for ads and embeds before they load
  • Avoid injecting content above existing content after page load
  • Preload custom fonts to avoid text shifting as fonts swap in

Why It Matters for SEO

Pages that pass the Core Web Vitals thresholds are eligible for Google's "Good page experience" designation. While Core Web Vitals are not the dominant ranking factor, they serve as a tiebreaker between pages of similar quality — and poor performance actively damages user experience and conversion rates regardless of rankings.
